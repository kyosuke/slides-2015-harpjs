## About Harp * 設定済みgulpのようなもの * 静的サイト生成 * zero-configuration web server with built in pre-processing
## 使い方 #### harpコマンドを入れる ``` $ sudo npm install -g harp ``` #### 初期データ生成 ``` $ harp init myproject ``` #### サーバ起動 ``` $ harp server myproject ``` ``` $ cd myproject $ harp server ```
## ルール
## 公開フォルダ ``` myapp.harp.io/ |- index.html <-- will be served ``` or ``` myapp.harp.io/ |- harp.json <-- required harp.json file |- README.md <-- won't be served |- secrets.txt <-- won't be served +- public/ <-- explicit public directory +- index.html <-- will be served ```
## アンダースコアで始まるファイルは公開されない ``` myapp.harp.io/ +- public/ |- index.html <-- will be served |- _some-partial.jade <-- won't be served +- _shared-partials/ <-- won't be served +- nav.jade ```
## 拡張子から自動で変換 ``` myfile.md -> myfile.html myfile.jade -> myfile.html myfile.ejs -> myfile.html myfile.less -> myfile.css myfile.styl -> myfile.css myfile.scss -> myfile.css myfile.sass -> myfile.css myfile.coffee -> myfile.js ```
## メタデータ ``` myapp.harp.io/ +- public/ |- index.jade +- articles/ |- _data.json <-- articles metadata goes here |- hello-world.jade <-- hello world article +- hello-brazil.jade <-- hello brazil article ```
## Harp Boilerplates https://github.com/harp-boilerplates ``` $ harp init myblog --boilerplate hb-blog $ harp init mydocs --boilerplate hb-docs ```
## Markdown 機能が足りない * GitHub Flavored Markdown * オレオレ拡張 * Jade内Markdown
